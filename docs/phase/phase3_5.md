## Phase 3.5: 코틀린 전환 안정화 및 구조 고도화

Java에서 Kotlin으로의 언어 전환 과정에서 발생한 기술적 부채를 해결하고, Kotlin의 언어적 특성을 활용하여 코드의 안정성과 가독성을 높였습니다.

#### 3.5-1. Kotlin 관용구(Idioms) 적용 및 컴파일 오류 해결

| 항목              | 세부 내용                                               | 상태 |
|-------------------|---------------------------------------------------------|-----|
| 타입 안전성 강화  | `GlobalExceptionHandler`의 Java Stream을 Kotlin `firstOrNull`로 대체 | ✅ |
| Nullability 처리  | Optional 대신 Kotlin의 Nullable 타입(`?`)과 Elvis 연산자(`?:`) 활용 | ✅ |
| 데이터 모델 개선  | `User.kt`의 Primary Constructor 및 Companion Object 활용 | ✅ |

---

#### 3.5-2. 아키텍처 구조 개선 (순환 참조 해결)

| 항목              | 세부 내용                                               | 상태 |
|-------------------|---------------------------------------------------------|-----|
| 설정 파일 분리    | `AuthConfig` 신설 — `PasswordEncoder` 빈 설정을 별도 분리 | ✅ |
| 의존성 체인 해소  | `SecurityConfig` ↔ `UserService` 간의 순환 참조 근본적 해결 | ✅ |
| Bean 관리 효율화  | 인증 관련 핵심 인프라와 보안 필터 설정의 책임 분리       | ✅ |

---

#### 3.5-3. 테스트 코드 안정화

| 항목              | 세부 내용                                               | 상태 |
|-------------------|---------------------------------------------------------|-----|
| Mocking 전략 수정 | `CustomOAuth2UserServiceTest`에서 생성자 기반 의존성 주입 대응 | ✅ |
| 테스트 호환성 확보| Java 기반 테스트 코드와 Kotlin 소스 코드 간의 상호운용성 검증 | ✅ |
