### Phase 3.5: 고가용성 대응 및 사용자 경험(UX) 고도화

상황<br/>
문제: 평소와 달리 이벤트로 인해 트래픽 폭주로 대기열이 발생했습니다. "당신은 500번째입니다"라는 문구는 사용자에게 거부감을 주고, 서비스가 멈춰있다는 인상을 줍니다.<br/>
목표: 사용자가 대기열에 있다는 사실을 인지하되, 그 시간이 '기다림'이 아닌 '유익한 탐색'이 되도록 설계합니다.

---
#### 1. Zero-Entry SSO 메커니즘
- 마이크로 콘텐츠 제공: 대기열 진입과 동시에 화면 하단 혹은 측면에 '오늘의 사내 주요 뉴스', '점심 메뉴', '동료들의 감사 인사' 등 가벼운 콘텐츠를 카드 슬라이드 형태로 제공합니다.
- 투명한 진행 상황: "500번째입니다"라는 정적인 문구 대신, **"인증 서버가 안전하게 준비 중입니다 (현재 85% 완료)"**와 같은 진행형 메시지와 애니메이션을 사용하여 심리적 대기 시간을 단축합니다.
- 논블로킹(Non-blocking) 진입: 대기가 완료되면 화면 전체가 새로고침되는 대신, 작업 중이던 화면 위로 토스트(Toast) 알림을 띄워 자연스러운 진입을 유도합니다.

#### 2. 비상 대응 메커니즘: 백그라운드 큐잉 (Background Queuing)

- 서버 리소스(Thread/DB Pool)가 임계치에 도달하면, 시스템은 즉시 **'차단 모드'**가 아닌 **'경험 유지 모드'**로 전환됩니다.
- Virtual Waiting Room: 사용자를 특정 '대기 페이지'에 가두지 않습니다. 메인 UI는 유지하되, 인증 프로세스만 비동기(Asynchronous)로 전환됩니다.
- Token-based Entry: 사용자가 로그인 시도 시 즉시 대기열 티켓(Redis Sorted Set 기반)을 발행하고, 클라이언트는 이 티켓을 가지고 백그라운드에서 서버와 통신(SSE 또는 Polling)합니다.

#### 3. 기대 효과 (The Impact)
- 시스템 생존성: 설정값을 변경하기 위해 서버를 재시작할 필요 없이, 기존 자원 내에서 시스템이 **'스스로를 보호(Self-Preservation)'**하며 가용성을 유지합니다.
- 브랜드 가치 상승: 갑작스러운 장애 상황에서도 당황하지 않고 세련된 UI/UX로 대응함으로써, 관공서 같은 딱딱한 시스템이 아닌 사용자를 배려하는 현대적인 서비스라는 인상을 심어줍니다.
- 업무 연속성: 사용자는 로그인을 기다리는 짧은 시간 동안에도 사내 정보를 습득할 수 있어, 시간 낭비 없는 업무 흐름을 유지할 수 있습니다.
