## Phase 1: 문제 정의 및 아키텍처 설계

Phase 1은 파편화된 서비스 운영 환경에서 발생하는 비효율을 진단하고, 이를 해결하기 위한 중앙 집중식 인증 체계의 초석을 다지는 단계입니다.

---

### 🔍 핵심 배경 및 문제 정의 (Core Scenarios)

#### 1. 관리자의 운영 스트레스 임계점 (Admin Experience)
회사 규모가 확장됨에 따라 서비스 A, B, C 등 다수의 프로젝트가 독립적으로 운영되고 있습니다. 각 서비스마다 관리자 계정을 별도로 생성하고 관리해야 하는 상황은 단순한 번거로움을 넘어 운영의 임계점에 도달했습니다.
- **문제**: 관리자가 업무 전환 시마다 로그인/로그아웃을 반복해야 하며, 이는 휴먼 에러(계정 혼동, 권한 오설정)의 주요 원인이 됨.
- **해결**: 중앙 집중식 인증 서버를 구축하여 **단 한 번의 로그인(SSO)**으로 모든 사내 서비스의 권한을 획득하는 사용자 경험(UX) 제공.

#### 2. 파편화된 회원 관리의 비효율 (Management Efficiency)
각 서비스가 독립된 DB와 회원 관리 로직을 보유함에 따라 전사 차원의 통합 통제가 불가능한 구조였습니다.
- **문제**: 퇴사자 발생이나 직무 변경 시 모든 서비스의 권한을 일일이 수정해야 함. 누락 발생 시 심각한 보안 취약점으로 직결됨.
- **해결**: 인증 서버를 **'Single Source of Truth(유일한 진실의 원천)'**로 정의. 한 곳에서의 권한 제어가 모든 연동 서비스에 즉시 반영되는 거버넌스 체계 구축.

#### 3. 무한한 확장성 및 오픈소스 연동 (Scalability)
신규 서비스 런칭이나 외부 오픈소스 툴(Grafana, Jenkins 등) 도입 시마다 인증 시스템을 중복 구현하는 낭비를 제거해야 합니다.
- **문제**: 새로운 도구 도입 시마다 '회원가입'부터 진행해야 하는 높은 진입 장벽과 관리 포인트 증가.
- **해결**: 글로벌 표준인 **OAuth2 프로토콜**을 준수하여, 어떤 신규 서비스나 오픈소스라도 설정만으로 즉시 사내 계정과 연동되는 '인증 생태계' 완성.

---

### 🏗️ Phase 1 설계 방향

| 항목 | 세부 내용 | 목표 |
|:---|:---|:---|
| **표준 프로토콜** | OAuth2 / OIDC 기반 설계 | 범용적인 확장성 및 보안성 확보 |
| **보안 강화** | Argon2 단방향 암호화 적용 | 사용자 비밀번호 보안 수준 극대화 |
| **인프라 기초** | PostgreSQL (User Entity) 설계 | 통합 회원 정보의 안정적 저장소 확보 |
| **인증 흐름** | Spring Security + JWT 기초 구현 | Stateless 인증 체계로의 전환 준비 |
