# 🏁 Roadmap to "Zero-Loading": 정복해야 할 기술적 고지들

본 문서는 티켓팅이나 선착순 이벤트와 같은 **폭발적 트래픽(Traffic Spike)** 상황에서도 사용자 로그인 지연을 0으로 만들기 위한 단계별 정복 지도입니다. 우리는 유행하는 기술을 쫓는 것이 아니라, 각 단계에서 마주한 **'물리적 한계'**를 깨부수며 전진합니다.

---

## 🏔️ High Ground 01: 트래픽 제어와 엔진 방어 (Phase 4)
*   **정복 목표**: "비정상 매크로 요청으로 인한 시스템 마비 차단"
*   **핵심 문제 (Problem)**: 정상 유저의 10배에 달하는 매크로가 입구를 막아 시스템 전체가 붕괴되는 현상.
*   **해결 가치 (Value)**: 비정상 요청을 즉각 도려내고, 수용량 초과 시 '가상 대기열'로 유저를 보호하여 **시스템 생존력(Resilience)** 확보.

## 🏔️ High Ground 02: 비동기 논블로킹 엔진으로의 도약 (Phase 5)
*   **정복 목표**: "쓰레드 점유로 인한 메모리 병목 및 I/O 대기 시간 박멸"
*   **핵심 문제 (Problem)**: 'Thread per Request' 방식은 수만 명의 동시 접속 시 쓰레드 생성 비용과 메모리 부족으로 인해 서버가 '먹통'이 됨.
*   **해결 가치 (Value)**: **Kotlin Coroutines & R2DBC** 도입으로 적은 자원으로 수만 개의 연결을 동시 처리. **하드웨어 효율성 10배 향상**.

## 🏔️ High Ground 03: 고가용성과 자가 치유 능력 (Phase 6)
*   **정복 목표**: "부분적 장애가 전체 시스템으로 번지는 연쇄 도미노 현상 방지"
*   **핵심 문제 (Problem)**: 외부 OAuth 서버나 DB 지연이 발생할 때, 인증 서버의 모든 쓰레드가 대기 상태에 빠져 전체 서비스가 중단되는 현상.
*   **해결 가치 (Value)**: **Circuit Breaker**와 **K8s HPA** 최적화를 통해 장애를 격리하고 트래픽에 따라 서버가 스스로 증식하는 **탄력적 구조** 완성.

## 🏔️ High Ground 04: 극한의 최적화와 Cold Start 0초 (Phase 7)
*   **정복 목표**: "서버 확장 속도와 런타임 성능의 물리적 한계 돌파"
*   **핵심 문제 (Problem)**: 트래픽 급증 시 서버가 새로 뜨는 데 걸리는 수 초의 시간(Cold Start) 동안 발생하는 초기 로딩.
*   **해결 가치 (Value)**: **GraalVM Native Image**로 서버 구동 시간을 0.1초대로 단축. 트래픽 폭주와 동시에 서버 인스턴스가 즉각 대응하는 **완벽한 로딩 0초** 달성.

---

## 🚀 Vision: 기술보다 '이유'
우리의 로드맵은 단순한 기능 구현의 목록이 아닙니다. **"어떤 폭풍 같은 트래픽 속에서도, 사용자가 '로그인' 버튼을 누르는 즉시 다음 화면을 마주하게 하겠다"**는 약속을 지키기 위한 처절한 응전의 기록입니다.
