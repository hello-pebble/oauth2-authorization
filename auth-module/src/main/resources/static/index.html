Base-Auth Portal

로그인은 가볍게, 서비스 보호는 확실하게.

시스템 아키텍처 가이드

본 포트폴리오는 "로그인은 언제나 가능해야 한다"는 철학으로 설계되었습니다.

1. 가벼운 인증 (Lightweight Auth)

인증 과정에서는 대기열이 발생하지 않습니다. 사용자는 즉시 JWT 토큰을 발급받아 자신의 정보를 확인할 수 있습니다.

2. 서비스별 보호 (Resource-based Protection)

부하가 큰 '매칭 서비스' 등 특정 리소스에 접근할 때만 실시간 트래픽을 분석하여 대기열 시스템(Virtual Waiting Room)이 작동합니다.

3. 게이트웨이 트래픽 제어

Spring Cloud Gateway가 각 요청의 헤더를 분석하여, 승인된 사용자만 백엔드 마이크로서비스로 라우팅합니다.

시나리오 테스트 방법

  1. 우측 폼을 통해 로그인을 진행합니다. (admin / password)
  2. 로그인 성공 후 나타나는 '매칭 서비스 진입' 버튼을 클릭합니다.
  3. 시스템 부하 상황을 시뮬레이션한 대기열 화면을 확인합니다.
  4. 순번이 되면 실제 서비스 API 결과가 프로필에 나타납니다.
